## M'identifier

J’ai besoin :

* **D’un mot de passe principal** pour mon coffre-fort (vaultwarden).
* **D’une identité unique** pour tous les services du cloud (kanidm / IDM).

Mon administrateur doit effectuer une procédure pour que je puisse définir ces accès.

* **Mot de passe principal** : un e-mail contenant un lien pour saisir le mot de passe.
* **Identité** : un QR-Code ou un lien pour définir une clé et un mot de passe "2FA".

:::tip[À savoir !]
- 🔒 **Mon mot de passe principal** (Vaultwarden / Bitwarden)<br/>permet de me connecter à mon coffre-fort.
- 🔑 **Une clé d’authentification** (Clé IDM ou [YubiKey](#utiliser-ma-yubikey))<br/>permet de me connecter à toutes les applications du cloud.
- 🔐 **Un 2ème facteur d'authentification** ([2FA](#ajouter-un-2ème-facteur))<br/>permet de renforcer un accès par mot de passe avec une étape supplémentaire.
:::

Le **mot de passe principal** est primordial, c'est le seul qu'il faut **absolument retenir**.

## Mes moyens de connexion

Mon compte accepte **quatre** moyens de me connecter aux services du cloud. Ils
coexistent&nbsp;: si l'un devient indisponible, les autres restent valables.

| Moyen | Ce que je fais | Quand |
|---|---|---|
| **YubiKey principale** | Code PIN de la clé, puis je la touche | Au quotidien |
| **YubiKey de secours** | Idem, clé rangée en lieu sûr | Si la principale est perdue |
| **Clé d'accès du coffre-fort** | Je déverrouille le coffre, je valide | Sans YubiKey sous la main |
| **Mot de passe + code 2FA** | Je tape le mot de passe, puis le code | Filet de sécurité |

:::tip[Je n'enregistre rien moi-même]
L'enregistrement des clés est fait **avec mon administrateur**. Je n'ai jamais à
créer une clé d'accès seul, ni à décider où elle se range.
:::

:::caution[Si la mauvaise fenêtre s'ouvre]
Je voulais utiliser ma YubiKey et c'est le coffre-fort qui s'ouvre&nbsp;? Ou
l'inverse&nbsp;? **Je refuse**, et je préviens mon administrateur. Valider la
mauvaise fenêtre peut remplacer une clé existante par une autre.
:::

## Utiliser ma YubiKey

Ma YubiKey est une clé physique qui remplace le mot de passe dans la plupart
des situations :

- **Ordinateur chiffré** : au démarrage, je touche la YubiKey quand la
  diode clignote → le disque se déverrouille tout seul, sans passphrase.
- **Connexion à ma session** : je touche la YubiKey sur l'écran de
  connexion au lieu de taper mon mot de passe.
- **Kanidm (SSO)** : l'enregistrement se fait une fois, avec mon
  administrateur. Ensuite, je saisis le code PIN de la clé et je la touche pour
  me connecter à tous les services du cloud.

:::note[Qu'est-ce qu'une clé d'accès ?]
Une clé d'accès permet de s'identifier **sans mot de passe**. Chaque clé est
rangée à **un seul endroit**&nbsp;: soit dans mon coffre-fort, soit sur ma
YubiKey. C'est ce rangement qui décide de ce qui s'ouvre au moment de me
connecter&nbsp;: la fenêtre du coffre-fort, ou la demande de code PIN de ma clé
physique.
:::

:::tip[YubiKey = le plus simple]
Une fois enregistrée, je n'ai plus qu'à toucher ma clé. Pas de mot de passe à
retenir ni de code à recopier.
:::

### Démarrer mon ordinateur chiffré

Si mon disque est chiffré, l'ordinateur demande à être déverrouillé au
démarrage, **avant** l'écran de connexion. C'est normal : tant que je ne l'ai
pas déverrouillé, son contenu est illisible — même en cas de vol.

- **Avec ma YubiKey** : je la branche avant d'allumer (ou dès que la demande
  apparaît), puis je la touche quand sa diode clignote. Le disque se
  déverrouille tout seul, rien à taper.
- **Sans la clé** : je tape la phrase secrète que mon administrateur m'a
  remise, puis <kbd>Entrée</kbd>. Les caractères ne s'affichent pas pendant la
  saisie, c'est voulu.
- **Rien ne fonctionne ?** Je peux me tromper et réessayer sans risque. Si je
  reste bloqué, je contacte mon administrateur : il peut déverrouiller ma
  machine **à distance**, sans se déplacer.

:::note[Phrase secrète ≠ mot de passe]
La phrase secrète du démarrage déverrouille le **disque** ; mon mot de passe
ouvre ensuite **ma session**. Ce sont deux choses différentes, et la YubiKey
peut remplacer les deux : un toucher au démarrage, un toucher à l'écran de
connexion.
:::

## Ajouter un 2ème facteur

On ajoute souvent un 2ème facteur à un mot de passe, pour le sécuriser.
De nos jours, énormément de mots de passe sont compromis, c’est pourquoi ce type
d’authentification devient nécessaire.

:::tip[Une authentification 2 facteurs (2FA) associe]
- Un mot de passe (que je connais)
- ET un autre “facteur” (que je possède)
:::

Le 2ème facteur peut être un code envoyé par mail / sms, un générateur de code à 6 chiffres, etc.

:::caution[Sécuriser mon coffre-fort Vaultwarden]
Un mot de passe principal “simple” est autorisé par défaut.
Mais il est vivement recommandé de lui adjoindre un 2ème facteur pour sécuriser l’accès,
surtout si je mets tous mes mots de passe dedans (banque, comptes sociaux, etc.).
:::
