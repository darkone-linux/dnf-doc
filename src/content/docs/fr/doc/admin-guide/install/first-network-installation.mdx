On met en place un réseau DNF **par étapes**, et on peut s'arrêter à chacune :
un poste suffit pour démarrer.

## Le parcours

1. **Un poste** : la machine de l'administrateur (cette page).
2. **Une passerelle** : créer une [zone](/fr/doc/admin-guide/install/gateway/) et y brancher d'autres postes.
3. **La coordination** : relier les zones par [VPN](/fr/doc/admin-guide/install/hcs/) (tailnet).

:::note[Une seule configuration]
Tous les nœuds sont décrits dans **une** configuration NixOS, sur la machine de
l'administrateur. De là, on déploie et fait évoluer tout le réseau.
:::

## Créer le projet

Le projet part du template `dnf-boilerplate` : une structure prête et un
`etc/config.yaml` commenté à remplir.

<Steps>

1. **Créer le dépôt**

   Depuis le template [dnf-boilerplate](https://github.com/darkone-linux/dnf-boilerplate)
   (bouton « Use this template »), ou par clone.

2. **Amorcer le framework**

   ```sh
   nix run github:darkone-linux/darkone-nixos-framework#init
   nix develop github:darkone-linux/darkone-nixos-framework
   ```

   :::note[Que fait `#init` ?]
   Il crée le lien `.dnf/` vers les recettes du framework. Ensuite,
   `just --list` affiche toutes les commandes disponibles.
   :::

3. **Initialiser les secrets**

   ```sh
   just configure-admin-host
   ```

   :::tip[Idempotent]
   Crée les clés age, le fichier `.sops.yaml` et un mot de passe par défaut.
   Ne recrée jamais ce qui existe déjà.
   :::

</Steps>

## Décrire la configuration

Tout se déclare dans `etc/config.yaml` (réseau, zones, utilisateurs, hôtes).
Pour un premier poste, un utilisateur et un hôte suffisent :

```yaml
# etc/config.yaml
users:
  alice:
    uid: 1000
    name: "Alice"
    profile: "nix-admin"
    groups: ["global"]

hosts:
  - hostname: "poste"
    name: "Poste de l'administrateur"
    profile: "desktop"
    users: ["alice"]
    disko:
      profile: "btrfs-1-disk"
      devices:
        main: "/dev/nvme0n1"
```

Puis régénérer et vérifier :

```sh
just generate      # produit var/generated/*.nix depuis etc/config.yaml
just check-flake   # nix flake check
```

:::caution[Fichiers générés]
`var/generated/` est produit par l'outil : on ne l'édite jamais à la main.
On édite `etc/config.yaml`, puis on régénère.
:::

## Installer le premier poste

L'installation grave une image, démarre la machine dessus, puis applique le
système complet.

<Steps>

1. **Générer l'image**

   ```sh
   just build-iso
   ```

2. **Démarrer la machine**

   Graver l'image sur une clé USB, brancher l'ordinateur au réseau, démarrer
   sur la clé, puis noter son adresse IP.

3. **Appliquer le système**

   ```sh
   # just full-install <hôte> <utilisateur> <ip>
   just full-install poste nix 10.0.0.50
   ```

   :::tip[En quelques minutes]
   `full-install` installe le système, les programmes, les configurations et les
   utilisateurs. En cas d'échec, les étapes détaillées sont dans le `Justfile`.
   :::

4. **Redémarrer** sans la clé.

</Steps>

:::note[On peut s'arrêter là]
Un poste autonome est déjà pleinement utilisable. La
[passerelle](/fr/doc/admin-guide/install/gateway/) n'est nécessaire que pour
créer une zone (réseau local partagé).
:::
