Le vocabulaire de base pour administrer un réseau DNF. Les pages suivantes y
font référence sans le redéfinir.

## Nix et NixOS

DNF repose entièrement sur [Nix / NixOS](https://nixos.org/). Quelques notions à
avoir en tête avant d'administrer le parc :

| Terme | Description |
|---|---|
| **Nix** | Gestionnaire de paquets **déclaratif** : on décrit l'état voulu, Nix le construit à l'identique. |
| **NixOS** | La distribution Linux bâtie sur Nix : **tout le système** (paquets, services, réseau) est décrit en code. |
| **Déclaratif** | On déclare *quoi* (le résultat), pas *comment* (les commandes) : le même code donne le même système. |
| **Reproductible** | Mêmes entrées → même système, sur n'importe quelle machine, à n'importe quel moment. |
| **Dérivation** | La « recette » de construction d'un élément (paquet, config), évaluée par Nix. |
| **Store** (`/nix/store`) | Dépôt **immuable** des éléments construits, identifiés par un hash. Rien ne s'écrase. |
| **Flake** | Unité de projet Nix avec entrées **épinglées** (`flake.lock`) : versions figées et reproductibles. |
| **Génération** | Un état complet du système, horodaté. Chaque déploiement en crée une nouvelle. |
| **Rollback** | Revenir à une génération précédente (au boot ou à chaud) : un déploiement raté n'est jamais définitif. |
| **Home Manager** | Gère de façon déclarative l'environnement d'un utilisateur (applis, bureau, dotfiles). |

<Aside type="tip" title="Pour aller plus loin">
DNF ajoute une couche au-dessus de NixOS : vous écrivez surtout du YAML
(`etc/config.yaml`), pas du Nix. La connaissance fine de Nix n'est utile que pour
[contribuer au framework](/fr/doc/dev-guide/).
</Aside>

## Le réseau

| Terme | Description |
|---|---|
| **Zone** | Sous-réseau autonome (LAN de la maison, réseau invité, site distant…). Préfixe IP en `/16`, une passerelle, ses hôtes. |
| **Nœud** | Toute machine du réseau : poste, serveur, passerelle, coordination. |
| **Passerelle** (*gateway*) | Nœud frontière d'une zone : routeur, pare-feu, DNS/DHCP, client VPN. Une par zone, IP fixe `<préfixe>.1.1`. |
| **Serveur de coordination** (*HCS*) | Nœud public qui orchestre le VPN maillé (Headscale) et héberge les services partagés entre zones. |
| **VPN** (*tailnet*) | La maille qui relie toutes les zones et les machines distantes, via Headscale + Tailscale. |

<Aside type="note" title="Vue d'ensemble">
Le schéma du réseau figure dans la [présentation du projet](/fr/doc/#le-réseau-dnf) ;
son fonctionnement détaillé est décrit dans [Réseau](/fr/doc/admin-guide/understand/network/).
</Aside>

## Hôtes, profils et services

| Terme | Description |
|---|---|
| **Hôte** | Nœud NixOS géré, déclaré dans `etc/config.yaml`. |
| **Profil d'hôte** | Gabarit qui définit le rôle d'un hôte (`gateway`, `server`, `desktop`, `laptop`, `portable`, `hcs`, `vm`…). Voir [Profils d'hôtes](/fr/doc/admin-guide/understand/host-profiles/). |
| **Profil d'utilisateur** | Environnement [Home Manager](https://github.com/nix-community/home-manager) d'un compte (`nix-admin`, `normal`, `student`, `child`…). |
| **Groupes** | Relient utilisateurs et hôtes : ils déterminent qui peut se connecter où. |
| **Services** | Briques auto-hébergées activées par hôte (Nextcloud, Immich, Forgejo…). Voir [Services](/fr/doc/admin-guide/operate/services/). |
| **Features** | Options non-service activées par hôte (`monitoring-node`, `nfs-client`…). |

## Framework, projet et configuration

Tout part d'un seul fichier déclaratif, transformé en configuration NixOS :

| Emplacement | Rôle |
|---|---|
| `etc/config.yaml` | La **source de vérité** : réseau, zones, utilisateurs, hôtes, services. |
| `dnf/` | Le **framework** : modules, profils, librairies. Réutilisable, peu modifié. |
| `usr/` | Le **projet local** (en écriture) : surcharges de modules et de profils, machines, secrets. |
| `var/generated/` | Le code Nix **généré** depuis `etc/config.yaml` (ne jamais éditer à la main). |
| `doc/` | Cette documentation. |

<Aside type="tip" title="Du YAML au système">
`etc/config.yaml` → `just generate` → `var/generated/*.nix` → framework.
Détail dans [Le générateur](/fr/doc/dev-guide/generator/) ; arborescence
complète dans l'[organisation des fichiers](/fr/doc/#organisation-des-fichiers).
</Aside>
