La supervision repose sur [Prometheus](https://prometheus.io/) (métriques),
[Grafana](https://grafana.com/) (tableaux de bord) et [Loki](https://grafana.com/oss/loki/)
(journaux). Elle est **organisée par zone** : un hôte collecte, les autres
exposent leurs métriques.

{/* t:p e758d17315b4c6280b72f038b192f1afbc448f15a250caedcca6cdeb1273a26e */}
## Les rôles

```d2
style.fill: transparent
direction: down

grafana: Hôte monitoring\nGrafana + Prometheus + Loki { shape: cylinder }
n1: Nœud\nnode-exporter { shape: rectangle }
n2: Nœud\nnode-exporter { shape: rectangle }
caddy: Hôte Caddy\nAlloy (logs) { shape: rectangle }

n1 -> grafana
n2 -> grafana
caddy -> grafana
```

| Rôle | Qui | Quoi |
|---|---|---|
| Collecteur | Hôte avec `prometheus` (+ `monitoring`, + `loki`) | Métriques & alertes, tableaux de bord, journaux |
| Nœud supervisé | Hôtes tagués `monitoring-node` | `node-exporter` (métriques système) |
| Source de logs | Hôtes sous Caddy | Alloy pousse les journaux d'accès |

`prometheus` (métriques + alertes) et `monitoring` (Grafana) sont **deux services
distincts** sur le même hôte ; Grafana requiert Prometheus. Le détail du
fonctionnement est dans
[Monitoring & Alertes](/fr/doc/admin-guide/understand/monitoring-alerts/).

:::note[Un collecteur par zone]
Un seul collecteur est admis par zone. Il supervise les nœuds dont le tag
`monitoring-node` désigne **sa** zone.
:::

## Activer

La supervision est un [service](/fr/doc/admin-guide/operate/services/). Sur le
collecteur :

```yaml
# etc/config.yaml
    services:
      prometheus:              # métriques + alertes (requis par monitoring)
      monitoring:
        domain: "stats"        # → stats.<zone>.domain.tld
```

- Les nœuds à superviser reçoivent le tag `monitoring-node` de leur zone.
- `loki` suit `monitoring` : les logs Caddy sont collectés **automatiquement**
  dès que Caddy et monitoring sont actifs.

:::tip[Accès]
Grafana et Prometheus sont protégés par le
[SSO Kanidm](/fr/doc/admin-guide/operate/sso/). La racine du domaine Grafana
redirige vers un tableau de bord d'accueil.
:::

:::note[Activer les alertes]
Les notifications Matrix/e-mail demandent un provisionnement unique du bot :
voir [Alertes](/fr/doc/admin-guide/operate/alerts/).
:::

## Réglages utiles

| Option | Effet | Défaut |
|---|---|---|
| `prometheus.retentionTime` | Rétention des métriques Prometheus | `30d` |
| `monitoring.isNode` | Fait de l'hôte un nœud supervisé | suit le tag |
| `monitoring.kioskTarget` | Tableau de bord affiché à la racine | node-exporter |

:::note[Référence]
Options détaillées : [module monitoring](/fr/ref/modules/service/) et le module
`loki` pour la collecte de journaux.
:::
