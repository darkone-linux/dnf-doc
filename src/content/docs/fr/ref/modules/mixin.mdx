### &#x1F4E6; darkone.host.desktop

Une configuration de bureau complète avec gnome, outils multimédias et outils bureautiques.

* **enable** `bool` Configuration d'hôte optimisée pour le bureau

```nix
darkone.host.desktop.enable = false;
```

### &#x1F4E6; darkone.host.gateway

La passerelle / routeur principal d'une zone de réseau local.

:::tip[Une passerelle prête à l'emploi!]
La passerelle est configurée dans le fichier `usr/config.yaml`.
Les services supplémentaires activés (homepage, adguardhome, forgejo, ncps...)
sont automatiquement configurés avec une infrastructure réseau cohérente sur la
passerelle et toutes les machines du réseau local.
:::

* **enable** `bool` Activer les fonctionnalités de passerelle pour l'hôte actuel (dhcp, dns, proxy, etc.)

```nix
darkone.host.gateway.enable = false;
```

### &#x1F4E6; darkone.host.hcs

Le serveur de coordination headscale principal.

:::tip[Un serveur headscale prêt à l'emploi!]
Le réseau est configuré dans le fichier `usr/config.yaml`.
Les services supplémentaires activés (authentification, etc.)
sont automatiquement configurés avec une infrastructure réseau cohérente sur votre
réseau global.

Alias Zsh "h" pour "headscale".
:::

* **enable** `bool` Activer le serveur de coordination headscale
* **enableClient** `bool` Activer le client tailscale sur le nœud HCS (recommandé pour héberger des services)

```nix
darkone.host.hcs = {
  enable = false;
  enableClient = true;
};
```

### &#x1F4E6; darkone.host.minimal

Chaque configuration d'hôte est basée sur cette configuration minimale.

:::caution[Déclaration de services]
Un certain nombre de services (immich, nextcloud, forgejo, etc.) peuvent être déclarés dans
`usr/config.yaml` de chaque hôte, quel que soit son type (serveur, ordinateur portable,
bureau, etc.). **Il est conseillé de les déclarer dans le fichier yaml pour que
le service soit visible sur l'ensemble du réseau!**
:::

* **enable** `bool` Configuration minimale de l'hôte
* **secure** `bool` Préférer des options plus sécurisées (désactiver les utilisateurs mutables...)

```nix
darkone.host.minimal = {
  enable = false;
  secure = false;
};
```

### &#x1F4E6; darkone.host.server

Configuration améliorée pour les serveurs (minimal, pas de veille, watchdog...).

:::caution[Le mode d'urgence est désactivé]
Nous préférons que le système tente de continuer le démarrage pour
que nous puissions toujours y accéder à distance. (cf. srvos)
:::

* **enable** `bool` Configuration minimale de l'hôte serveur
* **enableWatchdog** `bool` Activer le watchdog systemd

```nix
darkone.host.server = {
  enable = false;
  enableWatchdog = true;
};
```

### &#x1F4E6; darkone.profile.advanced

Profil pour les utilisateurs avancés (informaticiens, développeurs, administrateurs).

:::note[Profil d'hôte vs profil Home]
Ce module active des fonctionnalités pour tous les utilisateurs du système.
Utilisez les profils HomeManager pour fournir des logiciels adaptés à chaque utilisateur (admin, avancé, étudiant, enfant, adolescent, etc.).
:::

* **enable** `bool` Utilisateur avancé (admin sys, développeur)

```nix
darkone.profile.advanced.enable = false;
```
