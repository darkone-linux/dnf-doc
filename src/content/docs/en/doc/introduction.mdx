A **NixOS configuration** for multiple users, multiple hosts, and multiple services:

- 🔥 [Declarative, reproducible, immutable](https://nixos.org/).
- 🚀 [Modules](/en/ref/modules/) ready to use.
- ❄️ [Configuration](https://github.com/darkone-linux/darkone-nixos-framework/blob/main/usr/config.yaml) simplified.
- 🧩 [Organization](/en/doc/introduction/#structure) coherent.
- 🌎 A [complete network](#the-dnf-network).

:::note
This project evolves based on my needs. If you would like to be informed of upcoming stable versions, please let me know on [GitHub](https://github.com/darkone-linux/darkone-nixos-framework) or by subscribing to my [YouTube channel](https://www.youtube.com/@DarkoneLinux) (FR). Thank you!
:::

## Main Features

| | Feature | Description |
|---|---------|-------------|
| ⚙️ | Fully Automated | With [nixos-anywhere](https://github.com/nix-community/nixos-anywhere), [disko](https://github.com/nix-community/disko) and [colmena](https://github.com/zhaofengli/colmena) |
| 👤 | User Profiles | [Profiles](https://github.com/darkone-linux/darkone-nixos-framework/tree/main/dnf/home/profiles) and [modules](/en/ref/modules/#home-manager-modules) [Home Manager](https://github.com/nix-community/home-manager) (admin, gamer…) |
| 🖥️ | Host Profiles | [Host profiles](/en/ref/modules/#-darkonehostdesktop) (servers, network nodes, workstations…) |
| 🌐 | Tailnet VPN | [Mesh VPN](https://en.wikipedia.org/wiki/Mesh_network) with [headscale](https://headscale.net/) + [tailscale](https://tailscale.com/) + [subnets](#a-configuration-for-a-complete-network) |
| 🛡️ | Ad Blocker | Secure and ad-free internet with [AdguardHome](https://adguard.com/fr/adguard-home/overview.html) |
| 🧩 | Unique Identities | SSO with [Kanidm](https://kanidm.com/) and [Vaultwarden](https://github.com/dani-garcia/vaultwarden) |
| 🤗 | Smart Services | [Immich](https://immich.app/), [Nextcloud](https://nextcloud.com/), [Forgejo](https://forgejo.org/), [Matrix](https://matrix.com/), [Jellyfin](https://jellyfin.org/), [etc.](/en/ref/modules/#-darkoneserviceadguardhome) |
| 💻 | Clean GNOME | NixOS hosts with [GNOME](https://www.gnome.org/) and pre-configured apps |
| 💾 | 3-2-1 Backups | Robust, simplified, distributed [Restic](https://restic.net/) backups |
| 🏠 | Home Page | Automated home page for each zone |

## File Organization

At the root:

- `dnf` -> modules, users, hosts (framework)
- `usr` -> Local project (in write mode)
- `var` -> Generated files and logs
- `src` -> Generator source files
- `doc` -> Project documentation

### Structure

<FileTree collapsed={true}>
- flake.nix Project flake
- Justfile Project management with [just](https://github.com/casey/just)
- dnf/ Framework (modules & common files)
  - modules/ [Framework modules](/en/ref/modules/)
    - standard Standard modules
      - system/ System & Hardware
      - console/ CLI Application
      - graphic/ X Applications
      - service/ Network Services
      - admin/ Administration
      - user/ User configurations (outside HM)
    - mixin Macro-modules "Mixins"
      - host/ Host profiles (desktop, server...)
      - profile/ Complements to user profiles
  - home Home Manager configuration (HM)
    - modules/ Nix modules (features, programs)
    - profiles/ Profiles: admin, student, advanced...
    - nixos/ Additional NixOS configurations (outside HM)
- **usr/** Specific configuration for my network
  - config.yaml [My main configuration](/en/doc/specifications/#the-configuration-file) (en)
  - modules/ My NixOS modules, identical to `dnf/modules`
  - home/ My HM modules, identical to `dnf/home`
  - secrets/ My passwords
    - secret.yaml Passwords and SOPS keys
  - machines/ Specific configurations per host (hardware, etc.)
  - users/ Specific HM configurations per user
- var/ Generated files
  - log/ Log files
  - generated/ [Generated files](/en/doc/specifications/#the-generator) (en)
    - hosts.nix
    - users.nix
    - network.nix
- src/ Generator sources
- doc/ This documentation
</FileTree>

### Abstraction Layers

Lower levels serve upper levels.

![DNF Project layers](../../../../assets/layers.webp)

These configurations are organized by categories:

- **[Standard modules](/en/ref/modules/#standard-modules)** simple and easy to use.
- **[Mixin modules](/en/ref/modules/#mixin-modules)** that encompass multiple standard modules.
- **[User profiles](/en/doc/specifications/#create-a-user-profile-example)** (en), standard [Home Manager](https://github.com/nix-community/home-manager) configurations.
- **Tools for maintaining qualitative configuration.

## AI Usage

:::tip[Late Introduction of AI]
Until the end of April 2026, **the code for this project was entirely written by hand, without AI assistance**.
Because AI assistance will become essential to adapt to a new global development pace,
a policy has been in place since May 2026, preferring sovereign and local solutions.

- "Useful" code and architectural decisions -> **always manual**.
- Sharing the project into multiple smaller sub-projects.
- 100% testable code and reduction of side effects by AI.
- Project memory, context engineering, codebase index...
:::
