## Signing in

I need:

* **A master password** for my vault (Vaultwarden).
* **A unique identity** for all cloud services (Kanidm / IDM).

My administrator must perform a procedure so I can set up these credentials.

* **Master password**: an email containing a link to enter the password.
* **Identity**: a QR code or a link to set a key and a "2FA" password.

:::tip[Good to know!]
- 🔒 **My master password** (Vaultwarden / Bitwarden)<br/>allows me to connect to my vault.
- 🔑 **My authentication key** (IDM Key)<br/>allows me to connect to all cloud applications.
- 🔐 **My "2-factor" password** (2FA Key)<br/>also allows me to connect, if I have an issue with my key.
:::

The **master password** is essential, it's the only thing you **absolutely must remember**.

:::note[What is a key?]
A digital key allows you to authenticate without using a password.
It is stored in my vault.
When an application needs it, Bitwarden opens automatically.
It is the **SIMPLEST AND MOST SECURE** way to authenticate myself.
:::

:::note[A "2-factor" password?]
Two-factor authentication (2FA) uses:
- a password (something I know)
- AND another "factor" (something I have)

The latter can be a code sent by email / SMS, a 6-digit code generator, etc.
Nowadays, a huge number of passwords are compromised, which is why this type of authentication is becoming necessary.
:::

:::caution[Securing my vault]
To facilitate access to the vault (Vaultwarden / Bitwarden), a "simple" master password is allowed by default.
But it is strongly recommended to add a second factor to secure access, especially since I store all my passwords in it (bank, social accounts, etc.).
:::
