## Sign in

I need:

* **A master password** for my vault (vaultwarden).
* **A unique identity** for all cloud services (kanidm / IDM).

My administrator must follow a procedure so I can set these up.

* **Master password**: an email with a link to enter the password.
* **Identity**: a QR code or a link to set a key and a "2FA" password.

:::tip[Good to know!]
- 🔒 **My master password** (Vaultwarden / Bitwarden)<br/>lets me log into my vault.
- 🔑 **My authentication key** (IDM Key)<br/>lets me log into all cloud applications.
- 🔐 **My "2-factor" password** (2FA Key)<br/>also lets me log in if I have a problem with my key.
:::

The **master password** is essential, it's the only thing I **absolutely must remember**.

:::note[What is a key?]
A digital key lets me identify myself without using a password.
It is stored in my vault.
When an application needs it, Bitwarden opens automatically.
It is the **SIMPLEST AND MOST SECURE** way to authenticate.
:::

:::note[A "2-factor" password?]
A 2-factor authentication (2FA) uses:
- a password (something I know)
- AND another "factor" (something I have)

This can be a code sent by email/SMS, a 6-digit code generator, etc.
Nowadays, many passwords are compromised, which is why this type of authentication is becoming necessary.
:::

:::caution[Securing my vault]
To make accessing the vault (vaultwarden / bitwarden) easier, a "simple" master password is allowed by default.
But it is strongly recommended to add a second factor to secure access, especially since I store all my passwords in it (bank, social accounts, etc.).
:::
