The network hosts its own instant messaging, based on
[Matrix](https://matrix.org). Discussions, photo and file sharing,
audio and video calls&nbsp;: everything stays **with us**, on our server.

{/* t:p 03665ec3e120fd75051402c1ebdcbea1c95646d5c7e4b8b30a2f625ae5d8c95e */}
## What is Matrix?

Matrix is a messaging system comparable to WhatsApp or Telegram, with
three key differences&nbsp;:

- **Private**: messages are stored on the home server, not with
  a web giant.
- **Encrypted**: conversations are end-to-end encrypted by default.
- **Open**: you can choose [your application (client) among several](https://matrix.org/ecosystem/clients/),
  on computer or phone.

Conversations are organized into **rooms**&nbsp;: one room per topic, per
family, per project... and direct messages between two people.

## Connecting

My Matrix ID is built from my usual account&nbsp;:
`@prenom:domain.tld`. No new password to remember, authentication
uses the network's single sign-on (same as for other
services, see [Managing my account](/en/doc/user-guide/usage/account/)).

<Steps>

1. **On computer**: I open `https://element.domain.tld` in my
   browser, click **Sign in**, and authenticate with my
   network account.

2. **On phone**: I install the **Element** app (Android or
   iPhone). At startup, I choose "Other server" and enter
   `matrix.domain.tld`, then authenticate the same way.

3. On first connection, Element prompts me to **verify my session** and
   back up my encryption keys. I follow the instructions&nbsp;:
   this will let me read my old messages on a new
   device.

</Steps>

<Aside type="tip" title="The home domain name">
Throughout this page, `domain.tld` refers to the network's domain name.
If in doubt, ask the administrator.
</Aside>

## Bridges to WhatsApp, Telegram, and others

A **bridge** connects Matrix to an external messaging service. Once the bridge
is activated, my WhatsApp, Telegram, or Messenger conversations appear in
Element as regular rooms: I read and reply from Element, while my contacts see
no difference.

These bridges are built on the [mautrix](https://docs.mau.fi/bridges/go/setup.html)
project, the reference in the field. Each bridge has its own official documentation
page, linked in the sections below.

```d2
style.fill: transparent
direction: right

contact: My contact\nWhatsApp {
  shape: person
}
wa: WhatsApp
bridge: Bridge\n(home server) {
  shape: hexagon
}
element: Element\n(Matrix) {
  shape: rectangle
}
me: Me {
  shape: person
}

contact -> wa <-> bridge <-> element -> me
```

Each bridge is operated by a **bot**: a special contact I send commands to. The
principle is the same for all:

<Steps>

1. In Element, I start a **direct conversation** with the bridge bot (for
   example `@whatsappbot:domain.tld`).

2. I send it `login` and follow its instructions (scan a QR code, provide my
   phone number...).

3. That's it: my conversations appear gradually. Each user connects **their own
   account**, without interfering with other users on the network.

</Steps>

<Aside type="note" title="What about privacy?">
The bridge decrypts messages to relay them from one world to another: this is
unavoidable, but everything happens on **our server**, not in some unknown
cloud. If in doubt about a command, send `help` to the bot.
</Aside>

### WhatsApp

The WhatsApp bridge connects as a **linked device**, exactly like WhatsApp Web.
Simple and reliable.

<Steps>

1. Start a direct conversation with `@whatsappbot:domain.tld` and send it
   `login qr`.

2. On the phone: WhatsApp → Settings → **Linked devices** → Link a device,
   then scan the QR code displayed in Element.

</Steps>

<Aside type="tip" title="Keep the phone within reach">
Like WhatsApp Web, the phone must reconnect from time to time: beyond
**two weeks** offline, the linked device is disconnected and you need to
repeat the process. Alternative without scanning: `login phone` displays a
code to enter on the phone.
</Aside>

📖 [Official WhatsApp bridge documentation](https://docs.mau.fi/bridges/go/whatsapp/authentication.html)

### Signal

The Signal bridge connects as an additional **linked device**, without
touching the phone's existing connection.

<Steps>

1. Start a direct conversation with `@signalbot:domain.tld` and send it
   `login`.

2. On the phone: Signal → Settings → **Linked devices** → Add, then scan the
   QR code displayed in Element.

</Steps>

<Aside type="note" title="Message history">
When connecting, Signal may offer to **transfer conversation history**.
Otherwise, rooms fill up as new messages come in.
</Aside>

📖 [Official Signal bridge documentation](https://docs.mau.fi/bridges/go/signal/authentication.html)

### Telegram

The Telegram bridge connects with my **phone number**, like another Telegram
client.

<Steps>

1. Start a direct conversation with `@telegrambot:domain.tld` and send it
   `login phone +33612345678` (number in international format).

2. Enter the **verification code** sent by Telegram, then, if my account has a
   "two-step" password, that password.

</Steps>

<Aside type="note" title="Where does the code arrive?">
Telegram does **not send** the code by SMS: it displays it in an already open
Telegram session (phone or desktop). So you need an already connected Telegram
client. Alternative: `login qr` to scan a QR code.
</Aside>

📖 [Official Telegram bridge documentation](https://docs.mau.fi/bridges/go/telegram/authentication.html)

### Messenger (Facebook)

The Messenger bridge connects by reusing the **cookies** of a session open in a
browser. This is the most technical connection of all.

<Steps>

1. Start a direct conversation with `@messengerbot:domain.tld` and send it
   `login`.

2. Log in to `messenger.com` in a private window, then follow the bot's
   instructions to hand over the session cookies.

</Steps>

<Aside type="caution" title="Tricky step">
Retrieving cookies requires using the browser's **developer tools**. If in
doubt, ask the administrator for help. Enabling **two-factor authentication**
on the Facebook account limits blocks.
</Aside>

📖 [Official Meta (Messenger) bridge documentation](https://docs.mau.fi/bridges/go/meta/authentication.html)

### Discord

The Discord bridge connects with a simple QR code scan.

<Steps>

1. Start a direct conversation with `@discordbot:domain.tld` and send it
   `login-qr`.

2. Scan the QR code with the **Discord** app on your phone (Settings → Scan
   QR code).

</Steps>

<Aside type="caution" title="Optional bridge">
The Discord bridge is not enabled by default. If it does not respond, ask the
administrator to enable it.
</Aside>

📖 [Official Discord bridge documentation](https://docs.mau.fi/bridges/go/discord/authentication.html)

### Useful commands

Send to the bridge's bot, in its direct conversation&nbsp;:

| Command | Effect |
|---|---|
| `help` | List available commands |
| `login` | Connect my account |
| `logout` | Disconnect my account from the bridge |
