## Logging in

I need:

* **A master password** for my vault (vaultwarden).
* **A unique identity** for all cloud services (kanidm / IDM).

My administrator must follow a procedure so I can set up these credentials.

* **Master password**: an email containing a link to enter the password.
* **Identity**: a QR code or a link to set a key and a "2FA" password.

:::tip[Good to know!]
- 🔒 **My master password** (Vaultwarden / Bitwarden)<br/>allows me to log in to my vault.
- 🔑 **An authentication key** (IDM Key or [YubiKey](#using-my-yubikey))<br/>allows me to log in to all cloud applications.
- 🔐 **A 2nd authentication factor** ([2FA](#adding-a-second-factor))<br/>allows strengthening a password-based access with an additional step.
:::

The **master password** is essential, it's the only one you **must absolutely remember**.

## My login methods

My account accepts **four** ways to log in to cloud services. They coexist: if one becomes unavailable, the others remain valid.

| Method | What I do | When |
|---|---|---|
| **Main YubiKey** | Enter the key's PIN code, then touch it | Daily use |
| **Backup YubiKey** | Same, key stored in a safe place | If the main key is lost |
| **Vault access key** | Unlock the vault, confirm | When no YubiKey is at hand |
| **Password + 2FA code** | Enter the password, then the code | Safety net |

:::tip[Nothing is registered on my own]
Key registration is done **with my administrator**. I never have to create an access key on my own, or decide where to store it.
:::

:::caution[If the wrong window opens]
I wanted to use my YubiKey and the vault opens instead? Or the other way around? **I decline**, and I notify my administrator. Confirming the wrong window can replace an existing key with a different one.
:::

## Using my YubiKey

My YubiKey is a physical key that replaces the password in most situations:

- **Encrypted computer**: at boot, I touch the YubiKey when the LED blinks → the disk unlocks automatically, no passphrase needed.
- **Logging in to my session**: I touch the YubiKey on the login screen instead of typing my password.
- **Kanidm (SSO)**: registration is done once, with my administrator. After that, I enter the key's PIN code and touch it to log in to all cloud services.

:::note[What is an access key?]
An access key allows you to identify yourself **without a password**. Each key is stored in **one place only**: either in my vault, or on my YubiKey. This storage location determines what opens when I log in: the vault window, or the PIN prompt for my physical key.
:::

:::tip[YubiKey = simplest option]
Once registered, I just have to touch my key. No password to remember, no code to copy.
:::

## Adding a second factor

A second factor is often added to a password to secure it.
These days, a huge number of passwords are compromised, which is why this type
of authentication is becoming necessary.

:::tip[Two-factor authentication (2FA) combines]
- A password (something I know)
- AND another “factor” (something I have)
:::

The second factor can be a code sent by email / SMS, a 6-digit code generator, etc.

:::caution[Securing my Vaultwarden vault]
A “simple” master password is allowed by default.
But it is strongly recommended to add a second factor to secure access,
especially if I store all my passwords in it (bank, social accounts, etc.).
:::
