## Signing in

I need:

* **A master password** for my vault (Vaultwarden).
* **A unique identity** for all cloud services (Kanidm / IDM).

My administrator must perform a procedure so I can set up these access methods.

* **Master password**: an email containing a link to set the password.
* **Identity**: a QR Code or a link to set a key and a "2FA" password.

:::tip[Good to know!]
- 🔒 **My master password** (Vaultwarden / Bitwarden)<br/>allows me to log in to my vault.
- 🔑 **My authentication key** (IDM Key)<br/>allows me to log in to all cloud applications.
- 🔐 **My "2-factor" password** (2FA Key)<br/>also allows me to log in, if I have an issue with my key.
:::

The **master password** is essential, it's the only thing you **absolutely** need to remember.

:::note[What is a key?]
A digital key allows you to identify yourself without using a password.
It is stored in my vault.
When an application needs it, Bitwarden opens automatically.
It is the **SIMPLEST AND MOST SECURE** way to authenticate.
:::

:::note[A "2-factor" password?]
2-factor authentication (2FA) uses:
- a password (something I know)
- AND another "factor" (something I have)

This can be a code sent by email / SMS, a 6-digit code generator, etc.
Nowadays, many passwords are compromised, which is why this type of authentication is becoming necessary.
:::

:::caution[Securing my vault]
To make accessing the vault (Vaultwarden / Bitwarden) easier, a "simple" master password is allowed by default.
But it is highly recommended to add a 2nd factor to secure access, especially since I store all my passwords in it (bank, social accounts, etc.).
:::
