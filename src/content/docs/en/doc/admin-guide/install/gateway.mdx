A [zone](/en/doc/admin-guide/understand/concepts/) is a subnet, which has a **gateway**, behind which you can connect as many machines as you want.

{/* t:p 3172c667429340c38c55d1ae902aa5cd32c430372492e5375aa3f260b53a4a07 */}
## Hardware

The gateway is ideally installed on a mini-computer that is always on, with **at least two network interfaces**: one to the ISP box (WAN), the others to the local network (LAN, ethernet or wifi).

:::note[What hardware?]
For example, a [Protectli](https://eu.protectli.com/) box or equivalent:

- NVMe disk ≥ 256 GB
- coreboot BIOS preferably
- Optional wifi card
:::

## Declare the zone

The zone and its gateway are described in `etc/config.yaml`:

```yaml
# etc/config.yaml
zones:
  maison:
    description: "Home network"
    ipPrefix: "10.0"             # zone in 10.0.0.0/16
    gateway:
      wan:
        interface: "enp1s0"      # to the ISP box
      lan:
        interfaces: ["enp2s0"]   # to the local network

hosts:
  - hostname: "gw"
    name: "Gateway"
    zone: "maison:1.1"           # fixed IP 10.0.1.1
    profile: "gateway"
    services:
      adguardhome:               # DNS + ad blocking
      homepage:                  # service portal
    disko:
      profile: "server-1-disk"
      devices:
        main: "/dev/sda"
```

:::tip[Automatic services]
The `gateway` profile wires the router, firewall, DNS/DHCP (`dnsmasq`) and the
VPN client. Declared services (homepage, adguardhome…) are configured
automatically. See [Network](/en/doc/admin-guide/understand/network/).
:::

## Install the gateway

Installation is often done **remotely**: a person on site prepares the
machine, the administrator installs via SSH.

<Steps>

1. **Prepare the gateway** (on site)

   1. Plug the gateway into the box.
   2. Boot from the USB key (`just build-iso` image).
   3. Note the gateway's IP address.
   4. Redirect port 22 of the box to the gateway.

   :::note[Why redirect port 22?]
   It provides SSH access during installation. The USB key only accepts
   authentication keys (no password). The redirect can be removed
   afterward.
   :::

2. **Install** (administrator)

   A single command chains disk formatting, hardware configuration,
   deployment and cleanup&nbsp;:

   ```sh
   just full-install gw nixos <ip>
   ```

   - `gw`: hostname declared in `etc/config.yaml`.
   - `nixos`: user of the live USB key (DNF image).
   - `<ip>`: address noted in the previous step.

   :::note[The IP address may change]
   After the first reboot, the gateway may fall back to DHCP on a
   different address. While the script waits for it to return, you can
   **type the new IP** to continue without restarting everything.
   :::

</Steps>

## Register the gateway on the VPN

A gateway is a **Tailscale client**: it advertises its zone's subnet in
the [Headscale](/en/doc/admin-guide/operate/vpn/) mesh. Until it is
registered on the tailnet, other zones cannot see it.

Follow the dedicated procedure: [Register a
gateway](/en/doc/admin-guide/operate/vpn/#administration).
