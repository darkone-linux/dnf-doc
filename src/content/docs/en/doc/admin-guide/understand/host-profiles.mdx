A **host profile** is a template that defines the role of a machine. It is
chosen with `profile:` in `etc/config.yaml`. Profiles are inherited: each
starts from another and adds its specifics.

{/* t:p a3a2c805ce00c4f3bb8c1d8d0697ee3eb01508e32651337d6e8d01bfab6f56d9 */}

## Profiles

| Profile | Purpose | Inherits from | Provides |
|---|---|---|---|
| `minimal` | Base for every host | — | Hardware, firewall, i18n, zsh, sudo |
| `server` | Headless server | `minimal` | No sleep, watchdog, limited boot |
| `desktop` | Workstation | `minimal` | GNOME, multimedia, office |
| `laptop` | Laptop computer | `desktop` | Battery/sleep, sensors, thermald |
| `portable` | Bootable USB key | `minimal` | Extended hardware drivers |
| `gateway` | Zone gateway | `server` | Router, firewall, DNS/DHCP, VPN |
| `hcs` | VPN coordination | `server` | Headscale + shared services |
| `vm` | Virtual machine | `server` | Guest tools (VirtualBox/QEMU/Xen) |

## Choosing a profile

```yaml
# etc/config.yaml
hosts:
  - hostname: "poste"
    profile: "desktop"
```

:::tip[Services are independent of the profile]
A service (Immich, Nextcloud, Forgejo…) is declared per host **regardless** of
its profile. See [Services](/en/doc/admin-guide/operate/services/).
:::
