The commands below are all run with [`just`](https://just.systems/man/).

```sh title=Usage
just <command> <arguments>
```

Each command is prefixed with an icon indicating its context:

- рџ’  **Shared**: available both in the project and the framework.
- рџ”† **Framework**: specific to the upstream DNF framework (`dnf/`).
- вњіпёЏ **Project**: specific to the consuming project (my network).

:::caution[Privilege escalation]
Most commands run as the user administering the project, but some of them escalate privileges to the `nix` user (`sudo -u nix ...`).
:::

:::tip[Key commands]
Commands marked with a в­ђ are the most useful on a daily basis.
:::

## GIT Commands

These commands work on **all repositories** of the project (by default `src/*/`, `dnf`, `doc`, and the root). The short alias is `g`.

{/* t:p a1e6bad8fb415d7340e9d07849875119abef92d8dc270e84321e520836107748 */}
### вњіпёЏ `git <cmd> [project]`

Run any git command on one or all projects.

:::tip[Iterated projects]
Listed by `just git-show-repos` (all by default).
:::

```sh
just git "log --oneline -5"       # last commits of all projects
just git "status" dnf             # git status on dnf/ only
just git "remote -v"              # remotes of all projects
```

{/* t:p 48edf57ea025e134f58bc136457844f56ed64d9df6c39185730c9bd89dd2a4c8 */}
### вњіпёЏ `commit <message> [project]` в­ђ

`add -A` + `commit -m` on one or all projects. If the message is identical to the last commit, the recipe **amends** instead of creating a new commit.

```sh
just commit "fix(net): update dns config"
just commit "fix(net): update dns config" dnf    # dnf/ only
```

### вњіпёЏ `update-flake`

Updates flake inputs of `dnf/` then the root. If the last commit of `dnf/` is already an `update(flake):`, it amends it.

:::caution[Do not use in production]
`nix flake update` bumps all inputs to their latest version: only use for regular, tested updates.
:::

```sh
just update-flake
```

Syncs `flake.lock` with the local HEAD of `dnf/` (co-development).

:::note[No dnf/ checkout]
If `dnf/.git` does not exist (release mode), the command does nothing.
:::

```sh
just update-dnf
```

## Installation commands

Installing a new host follows a precise sequence.

{/* t:p d14baec5b0376679bae1d0a0a003046361405791774a8844fdc1752ef074921c */}
### вњіпёЏ `install <host> [user] [ip]`

Formats disks via **disko** and installs NixOS via **nixos-anywhere**.

:::danger[Destructive]
All data on the target host is wiped. Confirmation is required before execution.
:::

:::note[Requirements]
The host must have a disko configuration in `etc/config.yaml` and a `usr/machines/<host>/default.nix` directory.
:::

```sh

just install myserver do=test             # VM test only
```

## Deployment commands

The deployment tool is [colmena](https://github.com/zhaofengli/colmena).
All apply commands first refresh the DNF lock via
`update-dnf`.

{/* t:p 68a3a1962d58418a2c6ca2087c01b8a6f09b61aede53766186715eebd7c0eff9 */}
### вњіпёЏ `apply <target> [action]` / `a` в­ђ

Deploys the configuration to one or more targets.

| Parameter | Description |
|---|---|
| `target` | Hostname, pattern (`'*'`), list (`a,b`), colmena tag (`@server`) |
| `action` | `switch` (default), `boot`, `test`, `build` |

:::note[Build on target]
`--build-on-target`: compilation happens on the deployed host, not on the admin machine.
:::

```sh
just apply @all                 # all hosts
just apply myserver             # switch (default)
just apply myserver boot        # boot
just apply 'db-*'               # all hosts starting with db-
```

### вњіпёЏ `yubikey <user> [key] [action]`

Enrolls a YubiKey (`main` by default, or `backup`) for a user: PAM
(session/sudo), LUKS and Kanidm enrollment link, in a single pass on the
admin machine. `action=revoke` revokes the key across all machines at next
deployment. See [Strong authentication](/en/doc/admin-guide/operate/yubikey/).

```sh
just yubikey darkone            # enrolls the primary key
just yubikey darkone backup     # enrolls the backup key
just yubikey darkone main revoke
```

{/* t:p c721a639832b45fbdbb521264d00054fd9c54ef53c41e0d1c85d79114186406b */}
### вњіпёЏ `rotate init|push-keys|finalize`

Rotation of the age key `/etc/sops/age/infra.key` in 3 steps with integrity
checks.

| Step | Action |
|---|---|
| `init` | Generates a new key, keeps the old one, re-encrypts secrets |
| `push-keys` | Pushes the combined key (new + old) to all hosts |
| `finalize` | Verifies all hosts have the new key, removes the old one |

:::danger[Unfinished rotation]
Do not interrupt a rotation. Sidecars `.cur` / `.old` mark an ongoing rotation.
:::

```sh
just rotate init
just rotate push-keys
just rotate finalize
```

### вњіпёЏ `idm-sync-certs`

Collects Kanidm replication certificates from all fleet nodes
into `usr/secrets/replication/`.

:::note[Two-step deployment]
1. `just apply` : each node generates its identity.
2. `just idm-sync-certs` : collects the public certificates.
3. Commit + `just apply` : replication starts.
:::

```sh
just idm-sync-certs
```

Analyzes the health of Kanidm replication without modifying anything. Takes the
supplier node (HCS) and the list of consumers (gateways, comma-separated).

```sh

```
