## Reminder: level and category

```nix
darkone.system.security = {
  enable   = true;
  level    = "intermediary";   # minimal | intermediary | reinforced | high
  category = "server";         # base | client | server
};
```

Levels are **cumulative**: `intermediary` adds its rules to those of
`minimal`, and so on. The category adds the rules specific to a workstation
(`client`) or a server (`server`).

## Recommended category per profile

| Profile | Recommended category | Why |
|---|---|---|
| `minimal`, `portable` | `base` | Universal base |
| `desktop`, `laptop` | `client` | Screen, USB, interactive session |
| `server`, `gateway`, `hcs`, `vm` | `server` | Exposed services, network |

:::caution[Today: `base` everywhere]
Automatic profile вЖТ category mapping is not yet active. All hosts are
set to `minimal` / `base` by default. To apply a category, set
`category` manually on the relevant host.
:::

## Adapt without breaking everything

Two mechanisms allow bypassing a specific protection without lowering the overall
level.

- **Labels** (`excludes`): exclude a **group** of incompatible rules
  (e.g. `needs-jit`, `no-ipv6`).
- **Exceptions** (`exceptions`): override **one** rule, justification
  required.

Details of these mechanisms are in [Security
concepts](/en/doc/admin-guide/understand/security-notions/#exclusion-labels).

:::danger[Module being stabilized]
Raise the level **gradually** and test each host before production.
:::
