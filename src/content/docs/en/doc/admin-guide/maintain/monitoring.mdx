Monitoring relies on [Prometheus](https://prometheus.io/) (metrics),
[Grafana](https://grafana.com/) (dashboards) and [Loki](https://grafana.com/oss/loki/)
(logs). It is **organized by zone**: one host collects, the others
expose their metrics.

## Roles

```d2
style.fill: transparent
direction: down

grafana: Monitoring host\nGrafana + Prometheus + Loki { shape: cylinder }
n1: Node\nnode-exporter { shape: rectangle }
n2: Node\nnode-exporter { shape: rectangle }
caddy: Caddy host\nAlloy (logs) { shape: rectangle }

n1 -> grafana
n2 -> grafana
caddy -> grafana
```

| Role | Who | What |
|---|---|---|
| Collector | Host with `prometheus` (+ `monitoring`, + `loki`) | Metrics & alerts, dashboards, logs |
| Monitored node | Hosts tagged `monitoring-node` | `node-exporter` (system metrics) |
| Log source | Hosts behind Caddy | Alloy pushes access logs |

`prometheus` (metrics + alerts) and `monitoring` (Grafana) are **two separate
services** on the same host; Grafana requires Prometheus. Details are in
[Monitoring & Alerts](/en/doc/admin-guide/understand/monitoring-alerts/).

:::note[One collector per zone]
Only one collector is allowed per zone. It monitors nodes whose
`monitoring-node` tag points to **its** zone.
:::

## Enable

Monitoring is a [service](/en/doc/admin-guide/operate/services/). On the
collector:

```yaml
# etc/config.yaml
    services:
      prometheus:              # metrics + alerts (required by monitoring)
      monitoring:
        domain: "stats"        # → stats.<zone>.domain.tld
```

- Nodes to monitor receive the `monitoring-node` tag from their zone.
- `loki` follows `monitoring`: Caddy logs are collected **automatically** as
  soon as Caddy and monitoring are active.

:::tip[Access]
Grafana and Prometheus are protected by
[SSO Kanidm](/en/doc/admin-guide/operate/sso/). The Grafana domain root
redirects to a home dashboard.
:::

:::note[Enable alerts]
Matrix/email notifications require a one-time bot provisioning:
see [Alerts](/en/doc/admin-guide/operate/alerts/).
:::

## Useful settings

| Option | Effect | Default |
|---|---|---|
| `prometheus.retentionTime` | Prometheus metrics retention | `30d` |
| `monitoring.isNode` | Makes the host a monitored node | follows the tag |
| `monitoring.kioskTarget` | Dashboard displayed at the root | node-exporter |

:::note[Reference]
Detailed options: [monitoring module](/en/ref/modules/service/) and the `loki`
module for log collection.
:::
