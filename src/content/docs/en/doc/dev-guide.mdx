This section is for developers and AI agents who contribute to the framework:
understanding the architecture, creating or modifying features, and evolving the
documentation.

- [Project architecture](/en/doc/dev-guide/architecture/) — where things live, from abstraction layers down to files.
- [Modules](/en/doc/dev-guide/modules/) — how NixOS modules are organized and how to create a new one.
- [The generator](/en/doc/dev-guide/generator/) — from `config.yaml` to generated Nix code.
- [Contributing to the documentation](/en/doc/dev-guide/documentation/) — this Astro/Starlight site, its rules and tools.

:::note[AI policy]
Architecture decisions and "useful" code stay hand-written. AI agents assist with
writing, testing and documentation. See the
[project introduction](/en/doc/introduction/#ai-usage).
:::
