A user has lost access to their account. Before taking action, identify **which**
of the three passwords is involved: they are independent.

{/* t:p ae90d566c2871e797c39bb6e33a3100a567665d05f8e1144aaa74b9e6fd0d00d */}
## Three distinct passwords

| Password | Used for | Where to reset it |
|---|---|---|
| **Identity** (Kanidm) | Logging into services (SSO) | Kanidm token (below) |
| **Vault** (Vaultwarden) | Unlocking the password manager | Vaultwarden invitation |
| **Session** (NixOS) | Opening a session on a workstation | `just passwd` (below) |

{/* t:p 409d5473ebe1263a612303c621f549c028b7b690e407c1277131256d8d324cdd */}
## Identity and vault

<Steps>

1. **Vault (Vaultwarden)** — resend an invitation from the admin interface; the user resets their master password via email.
2. **Identity (Kanidm)** — generate a new reset token on the coordination server:

   ```bash
   just enter hcs
   kanidm person credential create-reset-token <login> --name idm_admin
   ```

3. Send the link or QR code to the user, who recreates their key and recovery password.

</Steps>

:::tip[Details]
The full account creation procedure is in the
[admin guide](/en/doc/admin-guide/install/hcs/#create-a-user-account).
:::

## Session password

The login password on a workstation is managed in
[secrets](/en/doc/admin-guide/operate/secrets/), not in Kanidm:

<Steps>

1. **Set the hash**

   ```bash
   just passwd <login>
   ```

2. **Commit and deploy**

   ```bash
   just commit "user(passwd): <login>"
   just apply @user-<login>
   ```

</Steps>

:::note[Default password]
When created, an account uses the fleet's default password
(`just passwd-default`). See [Users](/en/doc/admin-guide/operate/users/).
:::
